Xử Lý Sự Cố Thường Gặp

Dưới đây là một số vấn đề bạn có thể gặp phải khi triển khai hoặc vận hành TeleDrive và cách khắc phục.

1. Không truy cập được TeleDrive sau khi Deploy

Nguyên nhân:

  • Cổng (port) bị chặn bởi firewall.
  • TeleDrive chưa khởi động thành công trong Docker.
  • Sai địa chỉ IP hoặc cổng.

Cách khắc phục:

  1. Kiểm tra Firewall: Mở cổng mà TeleDrive đang chạy (ví dụ 8080) trên firewall của máy chủ.
    2. sudo ufw allow 8080/tcp
  2. Kiểm tra Docker Container:
    3. docker ps -a # Xem tất cả container
docker logs <container_id> # Xem log của container TeleDrive để tìm lỗi
  3. Kiểm tra địa chỉ: Đảm bảo bạn đang dùng đúng IP của máy chủ và đúng cổng.

2. Lỗi API_ID/API_HASH không hợp lệ

Nguyên nhân:

  • Nhập sai API_ID hoặc API_HASH trong file .env.
  • Telegram đã thu hồi key.

Cách khắc phục:

  1. Kiểm tra kỹ lại API_ID và API_HASH trên trang my.telegram.org.
  2. Đảm bảo không có khoảng trắng thừa.
  3. Khởi động lại TeleDrive sau khi sửa file .env:
    4. docker-compose down
docker-compose up -d

3. Lỗi đường dẫn ảnh ko tồn tại

Nguyên nhân:

  • Image ID (tên mã nguồn TeleDrive) được dùng trong file docker-compose.yml đã bị xóa Docker hub

Cách khắc phục:

  1. Tạo file Dockerfile để tự lấy 1 project teledrive khác trên github

4. Lỗi "Permission Denied" khi clone repo hoặc chạy lệnh

Nguyên nhân:

  • Bạn không có quyền truy cập vào thư mục.
  • File script không có quyền thực thi.

Cách khắc phục:

  1. Sử dụng sudo nếu cần (ví dụ: sudo git clone ...).
  2. Cấp quyền thực thi cho file nếu đó là script:
    3. chmod +x your_script.sh
